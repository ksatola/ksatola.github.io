<article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-a-gentle-introduction-to-computational-statistics" class="anchor" aria-hidden="true" href="#a-gentle-introduction-to-computational-statistics"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>A Gentle Introduction to Computational Statistics</h1>
<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p>There are two perspectives we can look at statistical problems while solving them: analytical and computational.</p>
<p><strong>Analytical methods</strong> are based on mathematics (mainly statistics and probability) and use of domain and theoretical knowledge. <strong>Computational methods</strong> are based on computer science (mainly software engineering) and use of fundamental computing techniques, like looping or random number generator.</p>
<p>Analytical methods were in many cases the only ones practically available in the past as data collection and computing power were expensive. Nowadays, computational methods become more and more attractive as data collection and computing gets cheaper and more easily available than before.</p>
<p>The main analytical tools are statistical tests and probability whereas computational approach is based on simulations.</p>
<p>In this paper, I will walk you through two examplary problems (simulating outcomes and permutation testing) and compare how they can be solved using both methods.</p>
<h2><a id="user-content-example-1-simulating-outcomes" class="anchor" aria-hidden="true" href="#example-1-simulating-outcomes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Example 1: Simulating Outcomes</h2>
<p>This example shows how we can do an efficient simulation of probabilities of common events and statistics.</p>
<p>Let's try to answer the following question using both methods: <strong>If you roll two 6-sided dice, how likely is it that the sum of results is greater than 7?</strong></p>
<h3><a id="user-content-analytical-method" class="anchor" aria-hidden="true" href="#analytical-method"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Analytical Method</h3>
<p>First, let's answer with the <strong>analytical method</strong>. The problem type above is variations with repetition. <code>We have 6 to the power of 2 possible outcomes, 36 pairs in total</code>. A pair (x, y) would consist of two numbers, x being a rolling result of the first die and y being a rolling result of the second one.</p>
<p>Our results would look like like these: (1, 1), (1, 2), (1, 3), ..., (6, 6) but we are interested in the ones where the sum of pair elements is greater than 7. Let's enumerate only the pair matching our requirement:</p>
<ul>
<li>(2, 6) -&gt; 1</li>
<li>(3, 5), (3, 6) -&gt; 2</li>
<li>(4, 4), (4, 5), (4, 6) -&gt; 3</li>
<li>(5, 3), ..., (5, 6) -&gt; 4</li>
<li>(6, 2), ..., (6, 6) -&gt; 5</li>
</ul>
<p>There are 15 pairs where the sum of elements is greater than 7. We can calculate the likelihood of getting the sum of results as:</p>
<p>&lt;number of pairs with the sum greater than 7&gt; / &lt;number of all possible pairs&gt; = 15/36 = 0.4166(6)</p>
<p><strong>The answer is:</strong> There is 42% of likelihood that if we roll two 6-sided dice we get the sum of the results greater than 7.</p>
<p>The example above is simple to solve analitically as the number of variations is reasonable small (36) but what if our problem was more complex? For example, how we could solve analitically the following question: <strong>If you roll seven 6-sided dice, how likely is it that the sum of results is greater than 35?</strong> In this case <code>we would have 6 to the power of 7 possible outcomes, which is 279,936 variations of pairs</code>. Solving this problem analitically would take us plenty of time...</p>
<h3><a id="user-content-computational-method" class="anchor" aria-hidden="true" href="#computational-method"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Computational Method</h3>
<p>Here is how we could answer the same question (<strong>If you roll two 6-sided dice, how likely is it that the sum of results is greater than 7?</strong>) with a simulation.</p>
<p>We will use <code>choices()</code> function from <code>random</code> module and <code>Counter</code> class from <code>collections</code> module.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">from</span> <span class="pl-s1">random</span> <span class="pl-k">import</span> <span class="pl-s1">choices</span>
<span class="pl-k">from</span> <span class="pl-s1">collections</span> <span class="pl-k">import</span> <span class="pl-v">Counter</span></pre></div>
<p>Then we define our die.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Define a 6-sided die</span>
<span class="pl-s1">faces</span> <span class="pl-c1">=</span> <span class="pl-en">list</span>(<span class="pl-en">range</span>(<span class="pl-c1">1</span>, <span class="pl-c1">7</span>))
<span class="pl-s1">faces</span>

[<span class="pl-c1">1</span>, <span class="pl-c1">2</span>, <span class="pl-c1">3</span>, <span class="pl-c1">4</span>, <span class="pl-c1">5</span>, <span class="pl-c1">6</span>]</pre></div>
<p>We will use <code>choices()</code> function to simulate a dice roll. The function takes our die definition as the first argument, and <code>k</code> defines a number of dice. If we only wanted we could also apply different weights to die faces. But not this time.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># A function for rolling a pair of dice</span>
<span class="pl-s1">roll_2_dice</span> <span class="pl-c1">=</span> (<span class="pl-k">lambda</span>: <span class="pl-en">choices</span>(<span class="pl-s1">population</span><span class="pl-c1">=</span><span class="pl-s1">faces</span>, <span class="pl-s1">weights</span><span class="pl-c1">=</span><span class="pl-c1">None</span>, <span class="pl-s1">k</span><span class="pl-c1">=</span><span class="pl-c1">2</span>))
<span class="pl-s1">pair</span> <span class="pl-c1">=</span> <span class="pl-en">roll_2_dice</span>()
<span class="pl-s1">pair</span>

[<span class="pl-c1">6</span>, <span class="pl-c1">5</span>]</pre></div>
<p>We can sum a pair of result using the <code>sum()</code> function.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-en">sum</span>(<span class="pl-s1">pair</span>)

<span class="pl-c1">11</span></pre></div>
<p>Now, the main part begins. We repeat the step described above 50,000 times and save the sums in the <code>rolls</code> list.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Simulate rolling a pair many times and tracking the outcomes</span>
<span class="pl-s1">rolls</span> <span class="pl-c1">=</span> [<span class="pl-en">sum</span>(<span class="pl-en">roll_2_dice</span>()) <span class="pl-k">for</span> <span class="pl-s1">_</span> <span class="pl-c1">in</span> <span class="pl-en">range</span>(<span class="pl-c1">50_000</span>)]</pre></div>
<p>We can use <code>Counter</code> class to return a dictionary, where the dictionary keys are sum values and the dictionary values are number of pairs with the same value of sum as the dictionary key.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Count those outcomes (number of sums of paired elements)</span>
<span class="pl-s1">rolls_counts</span> <span class="pl-c1">=</span> <span class="pl-v">Counter</span>(<span class="pl-s1">rolls</span>)
<span class="pl-en">print</span>(<span class="pl-s1">rolls_counts</span>)

<div class="highlight highlight-source-python"><pre><span class="pl-c"># Sort by faces [0] or counts [1] and store the sorted results as a dictionary</span>

<div class="highlight highlight-source-python"><pre><span class="pl-c"># If you roll two dice, how likely is it that your sum is greater than 7</span>

<h3><a id="user-content-analytical-vs-computational" class="anchor" aria-hidden="true" href="#analytical-vs-computational"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Analytical vs. Computational</h3>
<p>To summarize, here are the steps of the two approaches:</p>
<ul>
<li>
<p>Analytical approach</p>
<ul>
<li>Think about all possible outcomes</li>
<li>Enumerate outcomes which meet the condition (&gt;7)</li>
<li>Count the outcomes which meet the condition and calculate the likelihood</li>
</ul>
</li>
<li>
<p>Computational approach</p>
<ul>
<li>Define 6-sided die</li>
<li>Simulate rolling a pair of dice (many times)</li>
<li>Count and calculate the likelihood</li>
<li>Bonus: count, sort, visualize easily</li>
</ul>
</li>
</ul>
<p>Solving the problem analytically, we would have a problem with the more complex scenario (7 dice, sum greater than 35) whereas computationally we can solve it right away. The working code and the complex case solution can be found the Jupyter notebook linked below.</p>

<h2><a id="user-content-example-2-permutation-testing" class="anchor" aria-hidden="true" href="#example-2-permutation-testing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Example 2: Permutation Testing</h2>
<p>In this example <strong>we would like to know if a drug is effective or not</strong>. This is a classic A/B testing problem which we solve in an analytical and computational way.</p>
<p>The first and common to both methods step is to describe the problem mathematically and define a few inputs to our analysis. For this, we will use a standard hypothesis testing approach, defining our null and alternative hypotheses as follows:</p>
<p><a target="_blank" rel="noopener noreferrer" href="/ksatola/Computational-Statistics/blob/master/images/2019-04-12-002.png"><img src="/ksatola/Computational-Statistics/raw/master/images/2019-04-12-002.png" width="100%" style="max-width:100%;"></a></p>
<p>For the above, I will assume a significance level (alpha) of <code>0.01</code>. <strong>The significance level is the threshold for rejecting a difference between the groups</strong>.</p>
<p>In simple words, to confirm if the drug works, we need to find out if there is a <strong>statistically significant difference</strong> between the two groups of patients. We will be comparing means of the group results (samples) to see if they significantly differ or not.</p>
<p>We represent our groups as lists where each number is a result of patient's examination. In our case we do not know what the numbers means and if the higher number represents the better result or not. That is why two-tailed alternative hypothesis is chosen - we just want to confirm if the two groups differ significantly.</p>
<p>The <code>treatment group</code> is where patients get the drug and the <code>control group</code> is where patients do not get the drug (only a kind of mockup/not working replacement).</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">import</span> <span class="pl-s1">numpy</span> <span class="pl-k">as</span> <span class="pl-s1">np</span>

<span class="pl-c"># treatment group</span>
<span class="pl-s1">drug</span> <span class="pl-c1">=</span> [<span class="pl-c1">54</span>, <span class="pl-c1">73</span>, <span class="pl-c1">53</span>, <span class="pl-c1">70</span>, <span class="pl-c1">73</span>, <span class="pl-c1">68</span>, <span class="pl-c1">52</span>, <span class="pl-c1">65</span>, <span class="pl-c1">65</span>]

<span class="pl-c"># control group</span>

<span class="pl-c"># Significance level</span>

<p>For our groups, we observe <code>the means difference of 12.97</code> but the question is if this difference is statistically significant and the drug works? <span class="pl-s1">np</span>.<span class="pl-en">mean</span>(<span class="pl-s1">drug</span>) <span class="pl-c1">-</span> <span class="pl-s1">np</span>.<span class="pl-en">mean</span>(<span class="pl-s1">placebo</span>)

<h3><a id="user-content-analytical-method-1" class="anchor" aria-hidden="true" href="#analytical-method-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Analytical Method</h3>
<p>First, let's try to answer to this question analytically. For that <strong>we will use T-Student sampling distribution and calculate t-test statistic</strong>. Statistical tests usually come with some assumptions, and t-test is not an exception.</p>
<p>T-test assumptions:</p>
<ol>
<li>Both groups (drug and placebo) should be normally distributed.</li>
<li>Samples should come from populations with equal variances.</li>
<li>Samples should be of the same size.</li>
<li>At best there should be more than 20 observations in a sample.</li>
</ol>
<p>Assumptions 3 and 4 are quick to deal with. Our first sample (drug) has 9 observations and the second (placebo) 10. They are not equal in size but close, so we can live with it. The small number of observations (less than 20) may impact the quality of t-test outcome but we have no other choice but to try (we do not have any other observations).</p>
<p>To check if assumption one is met, we will use <strong>Shapiro-Wilk test</strong> designed specifically for samples containing less than 50 observations. For that we will use a helper function <code>normaltest()</code> which utilizes <code>shapiro()</code> function of the <code>scipy.stats</code> module.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">from</span> <span class="pl-s1">scipy</span> <span class="pl-k">import</span> <span class="pl-s1">stats</span>

<span class="pl-k">def</span> <span class="pl-en">normaltest</span>(<span class="pl-s1">sample</span>, <span class="pl-s1">name</span>, <span class="pl-s1">alpha</span><span class="pl-c1">=</span><span class="pl-c1">0.05</span>):
<span class="pl-s">'''</span>
<span class="pl-s"> Shapiro-Wilk test for samples with n &lt; 50 observations.</span>
<span class="pl-s"> Reference: https://docs.scipy.org/doc/scipy/reference/generated/scipy.stats.shapiro.html</span>
<span class="pl-s"> '''</span>
<span class="pl-v">W</span>, <span class="pl-s1">p</span> <span class="pl-c1">=</span> <span class="pl-s1">stats</span>.<span class="pl-en">shapiro</span>(<span class="pl-s1">sample</span>)
<span class="pl-c">#print("W = {:g}".format(W))</span>
<span class="pl-c">#print("p = {:g}".format(p))</span>
<span class="pl-k">if</span> <span class="pl-s1">p</span> <span class="pl-c1">&lt;</span> <span class="pl-s1">alpha</span>: <span class="pl-c"># null hypothesis: sample comes from a normal distribution</span>
<span class="pl-en">print</span>(<span class="pl-s">"{} does not come from a normal distribution"</span>.<span class="pl-en">format</span>(<span class="pl-s1">name</span>))
<span class="pl-k">else</span>:
<span class="pl-en">print</span>(<span class="pl-s">"{} comes from a normal distribution"</span>.<span class="pl-en">format</span>(<span class="pl-s1">name</span>))</pre></div>
<p>Let's now test if our first assumption is true regarding the drug and placebo samples.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Is drug normally distributed?</span>
<span class="pl-en">normaltest</span>(<span class="pl-s1">drug</span>, <span class="pl-s">"Drug"</span>)

<span class="pl-v">Drug</span> <span class="pl-s1">comes</span> <span class="pl-k">from</span> <span class="pl-s1">a</span> <span class="pl-s1">normal</span> <span class="pl-s1">distribution</span></pre></div>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Is placebo normally distributed?</span>
<span class="pl-en">normaltest</span>(<span class="pl-s1">placebo</span>, <span class="pl-s">"Placebo"</span>)

<span class="pl-v">Placebo</span> <span class="pl-s1">comes</span> <span class="pl-k">from</span> <span class="pl-s1">a</span> <span class="pl-s1">normal</span> <span class="pl-s1">distribution</span></pre></div>
<p>Our samples come from a normal distribution, although it is difficult to say that by just looking at data because we have only 9 and 10 observations.</p>
<p>Now it is time to deal with the second assumtion, stating that our samples should come from populations with equal variances. It is important to mention that we are not checking variances of the samples (their variances are not equal) but if they come from populations with equal variances.</p>
<p>To check if the second assumption is valid, we will use <code>varsequals()</code> helper function utilizing <code>bartlett()</code> function from <code>scipy.stats</code> module. The function uses <strong>Bartlett's test</strong> which answers the question if our samples come from populations with equal variances.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">def</span> <span class="pl-en">varsequals</span>(<span class="pl-s1">sample1</span>, <span class="pl-s1">sample2</span>, <span class="pl-s1">alpha</span><span class="pl-c1">=</span><span class="pl-c1">0.01</span>):
<span class="pl-s">'''</span>
<span class="pl-s"> Perform Bartlettâs test for equal variances.</span>
<span class="pl-s"> Bartlettâs test tests the null hypothesis that all input samples are from populations with equal variances.</span>
<span class="pl-s"> Reference: https://docs.scipy.org/doc/scipy-0.14.0/reference/generated/scipy.stats.bartlett.html</span>
<span class="pl-s"> '''</span>
<span class="pl-v">T</span>, <span class="pl-s1">p</span> <span class="pl-c1">=</span> <span class="pl-s1">stats</span>.<span class="pl-en">bartlett</span>(<span class="pl-s1">sample1</span>, <span class="pl-s1">sample2</span>)
<span class="pl-k">if</span> <span class="pl-s1">p</span> <span class="pl-c1">&lt;</span> <span class="pl-s1">alpha</span>: <span class="pl-c"># null hypothesis: all input samples are from populations with equal variances</span>
<span class="pl-en">print</span>(<span class="pl-s">"Not all input samples are from populations with equal variances."</span>)
<span class="pl-k">else</span>:
<span class="pl-en">print</span>(<span class="pl-s">"All input samples are from populations with equal variances."</span>)</pre></div>
<p>It seems that our second assumption is also valid:</p>
<div class="highlight highlight-source-python"><pre><span class="pl-en">varsequals</span>(<span class="pl-s1">drug</span>, <span class="pl-s1">placebo</span>)

<span class="pl-v">All</span> <span class="pl-s1">input</span> <span class="pl-s1">samples</span> <span class="pl-s1">are</span> <span class="pl-k">from</span> <span class="pl-s1">populations</span> <span class="pl-k">with</span> <span class="pl-s1">equal</span> <span class="pl-s1">variances</span>.</pre></div>
<p>Now, we can perform our test and check if the mean difference of our two samples is statistically significant. The remaining question is if our samples are expected to have been drawn from the same population. To do this, we will do a <strong>t-test</strong> using <code>ttest_ind()</code> function from <code>scipy.stats</code> module.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># We assume all observations are independent</span>
<span class="pl-s1">t_stat</span>, <span class="pl-s1">p</span> <span class="pl-c1">=</span> <span class="pl-s1">stats</span>.<span class="pl-en">ttest_ind</span>(<span class="pl-s1">drug</span>, <span class="pl-s1">placebo</span>)
<span class="pl-en">print</span>(<span class="pl-s">'t={:.4f}, p={:.4f}'</span>.<span class="pl-en">format</span>(<span class="pl-s1">t_stat</span>, <span class="pl-s1">p</span>))

<span class="pl-s1">t</span><span class="pl-c1">=</span><span class="pl-c1">3.9357</span>, <span class="pl-s1">p</span><span class="pl-c1">=</span><span class="pl-c1">0.0011</span></pre></div>
<p>The <code>t-test statistic value is 3.9357</code> and our <code>p-value is 0.0011</code>. The <strong>p-value</strong> is a probability of obtaining a result equal to or more extreme than was observed in the data. If our p-value (0.0011) is less than our alpha (0.01 / 2 = 0.005) than we can reject our null hypothesis (that there is no difference between drug and placebo) and assume the alternative hypothesis that the drug is effective.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Interpret via p-value</span>
<span class="pl-k">if</span> <span class="pl-s1">p</span> <span class="pl-c1">&lt;</span> <span class="pl-s1">alpha</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Reject the null hypothesis that the means are equal.'</span>)
<span class="pl-k">else</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Accept null hypothesis that the means are equal.'</span>)
<span class="pl-v">Reject</span> <span class="pl-s1">the</span> <span class="pl-s1">null</span> <span class="pl-s1">hypothesis</span> <span class="pl-s1">that</span> <span class="pl-s1">the</span> <span class="pl-s1">means</span> <span class="pl-s1">are</span> <span class="pl-s1">equal</span>.</pre></div>
<p>The p-value returned by the <code>ttest_ind()</code> implementation of t-test is two-tailed by design (if we were testing one-tailed hypothesis, is such case we would have to divide the p-value by 2). Again, in our case the p-value is far lower than our alpha, so we can conclude that we can reject the null hypothesis.</p>
<p>We can also confirm our finding calculating the <code>degrees of freedom</code>,</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Calculate degrees of freedom</span>
<span class="pl-s1">df</span> <span class="pl-c1">=</span> <span class="pl-en">len</span>(<span class="pl-s1">drug</span>) <span class="pl-c1">+</span> <span class="pl-en">len</span>(<span class="pl-s1">placebo</span>) <span class="pl-c1">-</span> <span class="pl-c1">2</span>
<span class="pl-s1">df</span>

<span class="pl-c1">17</span></pre></div>
<p>and the <code>critical value</code>.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Calculate the critical value (two-tailed test)</span>
<span class="pl-c"># PPF (percent point function)</span>
<span class="pl-s1">cv</span> <span class="pl-c1">=</span> <span class="pl-s1">stats</span>.<span class="pl-s1">t</span>.<span class="pl-en">ppf</span>(<span class="pl-c1">1.0</span> <span class="pl-c1">-</span> <span class="pl-s1">alpha</span><span class="pl-c1">/</span><span class="pl-c1">2</span>, <span class="pl-s1">df</span>)
<span class="pl-s1">cv</span>

<span class="pl-c1">2.8982305196347173</span></pre></div>
<p>Using the critical value and t-test statistic we infer the same as above. Our drug works.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Interpret via critical value (abs for symmetric distribution)</span>
<span class="pl-k">if</span> <span class="pl-en">abs</span>(<span class="pl-s1">t_stat</span>) <span class="pl-c1">&lt;=</span> <span class="pl-s1">cv</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Accept null hypothesis that the means are equal.'</span>)
<span class="pl-k">else</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Reject the null hypothesis that the means are equal.'</span>)
<span class="pl-v">Reject</span> <span class="pl-s1">the</span> <span class="pl-s1">null</span> <span class="pl-s1">hypothesis</span> <span class="pl-s1">that</span> <span class="pl-s1">the</span> <span class="pl-s1">means</span> <span class="pl-s1">are</span> <span class="pl-s1">equal</span>.</pre></div>
<p><strong>To summarize the analytical approach</strong>, we proved that we have sufficient evidence to reject the null hypothesis which means that the drug works.</p>
<h3><a id="user-content-computational-method-permutation-testing" class="anchor" aria-hidden="true" href="#computational-method-permutation-testing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Computational Method (Permutation Testing)</h3>
<p>Now it is time to check if the drug is effective in a computational way. We will be using our hypothesis definition but instead of doing a statistical test and satisfying its assumption we assume the null hypothesis is true, so there is no difference in the groups.</p>
<p>To refresh, the observed mean difference in drug and placebo groups is 12.97 (observed_diff).</p>
<div class="highlight highlight-source-python"><pre><span class="pl-s1">observed_diff</span> <span class="pl-c1">=</span> <span class="pl-en">mean</span>(<span class="pl-s1">drug</span>) <span class="pl-c1">-</span> <span class="pl-en">mean</span>(<span class="pl-s1">placebo</span>)
<span class="pl-en">print</span>(<span class="pl-s">f"<span class="pl-s1"><span class="pl-kos">{</span><span class="pl-s1">observed_diff</span>:.2f<span class="pl-kos">}</span></span>"</span>)

<span class="pl-c1">12.97</span></pre></div>
<p>If we assume that there is no difference in means of the treatment and control groups, we can merge the observations from both groups and then check how likely is that the observed_diff value or greater appears (we will calculate an absolute value of observed_diff to cover two-tailed case).</p>
<p>We will now simulate the mean differences doing random groups assignment from the common pool (drug and placebo combined) many times.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-k">from</span> <span class="pl-s1">statistics</span> <span class="pl-k">import</span> <span class="pl-s1">mean</span>
<span class="pl-k">from</span> <span class="pl-s1">random</span> <span class="pl-k">import</span> <span class="pl-s1">shuffle</span>

<span class="pl-c"># Unite the data</span>
<span class="pl-s1">combined</span> <span class="pl-c1">=</span> <span class="pl-s1">drug</span> <span class="pl-c1">+</span> <span class="pl-s1">placebo</span>

<span class="pl-s1">n</span> <span class="pl-c1">=</span> <span class="pl-c1">10_000</span>
<span class="pl-s1">count</span> <span class="pl-c1">=</span> <span class="pl-c1">0</span>
<span class="pl-s1">simulated_means</span> <span class="pl-c1">=</span> []

<span class="pl-c"># Simulate it a bunch of times</span>
<span class="pl-k">for</span> <span class="pl-s1">_</span> <span class="pl-c1">in</span> <span class="pl-en">range</span>(<span class="pl-s1">n</span>):
<span class="pl-c"># Rearrange in-place</span>
<span class="pl-en">shuffle</span>(<span class="pl-s1">combined</span>)
<span class="pl-c"># Split combined into simulated drug and placebo grups</span>
<span class="pl-c"># and calculate a simulated mean difference</span>
<span class="pl-s1">shuffled_diff</span> <span class="pl-c1">=</span> <span class="pl-en">mean</span>(<span class="pl-s1">combined</span>[:<span class="pl-en">len</span>(<span class="pl-s1">drug</span>)]) <span class="pl-c1">-</span> <span class="pl-en">mean</span>(<span class="pl-s1">combined</span>[<span class="pl-en">len</span>(<span class="pl-s1">drug</span>):])
<span class="pl-c"># Add the difference for future reference</span>
<span class="pl-s1">simulated_means</span>.<span class="pl-en">append</span>(<span class="pl-s1">shuffled_diff</span>)
<span class="pl-c"># Increase counter if the simulated mean difference</span>
<span class="pl-c"># is equal or greater from the observed mean difference</span>
<span class="pl-c"># abs() as we test two-tailed alternative hypothesis</span>
<span class="pl-s1">count</span> <span class="pl-c1">+=</span> (<span class="pl-en">abs</span>(<span class="pl-s1">shuffled_diff</span>) <span class="pl-c1">&gt;=</span> <span class="pl-s1">observed_diff</span>)</pre></div>
<p>If the null hypothesis is true (the drug does not work, on average group means are close to 0) we should observe many occurences of mean differences equal or greater to observed_diff. If not, then we can safely reject the null hypothesis (as the observed_diff would be exceptional for samples drawn from the same distribution).</p>
<p>Our simulation result shows that 16 times on 10,000 simulated mean difference was equal or greater then observed one (every time we run out test we may get a little bit different number). According to our logic, if such difference was common we would get the difference at least 12.97 many more times.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-en">print</span>(<span class="pl-s">f"""<span class="pl-s1"><span class="pl-kos">{</span><span class="pl-s1">n</span>:,<span class="pl-kos">}</span></span> label reshufflings produced only <span class="pl-s1"><span class="pl-kos">{</span><span class="pl-s1">count</span><span class="pl-kos">}</span></span> instances </span>
<span class="pl-s">with a difference at least as extreme as the observed difference of <span class="pl-s1"><span class="pl-kos">{</span><span class="pl-s1">observed_diff</span>:.2f<span class="pl-kos">}</span></span>."""</span>)

<span class="pl-c1">10</span>,<span class="pl-c1">000</span> <span class="pl-s1">label</span> <span class="pl-s1">reshufflings</span> <span class="pl-s1">produced</span> <span class="pl-s1">only</span> <span class="pl-c1">16</span> <span class="pl-s1">instances</span>
<span class="pl-k">with</span> <span class="pl-s1">a</span> <span class="pl-s1">difference</span> <span class="pl-s1">at</span> <span class="pl-s1">least</span> <span class="pl-k">as</span> <span class="pl-s1">extreme</span> <span class="pl-k">as</span> <span class="pl-s1">the</span> <span class="pl-s1">observed</span> <span class="pl-s1">difference</span> <span class="pl-s1">of</span> <span class="pl-c1">12.97</span>.</pre></div>
<p>We can also calculate a <strong>p-value</strong> easily. To recall, <strong>p-value</strong> is a chance of observing the current difference when there is truly no difference. In other words it is a probability of obtaining a result equal to or more extreme than was observed in the data.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-s1">p</span> <span class="pl-c1">=</span> <span class="pl-s1">count</span> <span class="pl-c1">/</span> <span class="pl-s1">n</span>
<span class="pl-s1">p</span>

<span class="pl-c1">0.0016</span></pre></div>
<p>The p-value is only 0.0016 and is far less than our significance level (alpha) established at 0.01/2 = 0.005 level. It is important to notice, that this p-value value has nothing to do with the p-value calculated with the analytical method.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-c"># Interpret via p-value</span>
<span class="pl-k">if</span> <span class="pl-s1">p</span> <span class="pl-c1">&lt;</span> <span class="pl-s1">alpha</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Reject the null hypothesis that the means are equal.'</span>)
<span class="pl-k">else</span>:
<span class="pl-en">print</span>(<span class="pl-s">'Accept null hypothesis that the means are equal.'</span>)
<span class="pl-v">Reject</span> <span class="pl-s1">the</span> <span class="pl-s1">null</span> <span class="pl-s1">hypothesis</span> <span class="pl-s1">that</span> <span class="pl-s1">the</span> <span class="pl-s1">means</span> <span class="pl-s1">are</span> <span class="pl-s1">equal</span>.</pre></div>
<p>One of the nice "side effects" of computational approach is that doing simulations we gather data we can use to interpret our result visually.</p>
<div class="highlight highlight-source-python"><pre><span class="pl-s1">simulated_means</span> <span class="pl-c1">=</span> <span class="pl-s1">np</span>.<span class="pl-en">asarray</span>(<span class="pl-s1">simulated_means</span>)
<span class="pl-s1">plt</span>.<span class="pl-en">figure</span>(<span class="pl-s1">figsize</span><span class="pl-c1">=</span>(<span class="pl-c1">16</span>, <span class="pl-c1">8</span>))

<h3><a id="user-content-analytical-vs-computational-1" class="anchor" aria-hidden="true" href="#analytical-vs-computational-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Analytical vs. Computational</h3>
<p>The analytical and computational approaches are in general very similar except the core part (in bold below):</p>
<ul>
<li>
<p>Analytical approach</p>
<ul>
<li>Setup testing framework.
<ul>
<li>Define null and alternative hypothesis.</li>
<li>Set alpha (the threshold for rejecting differences).</li>
</ul>
</li>
<li>Collect data.</li>
<li><strong>Pick a sampling distribution, calculate a test statistic, remember about assumptions.</strong></li>
<li>Calculate p-value <strong>(and if needed, also the critical value)</strong>.</li>
<li>Draw conclusion.</li>
</ul>
</li>
<li>
<p>Computational approach</p>
<ul>
<li>Setup testing framework.
<ul>
<li>Define null and alternative hypothesis.</li>
<li>Set alpha (the threshold for rejecting differences).</li>
</ul>
</li>
<li>Collect data.</li>
<li><strong>Iterate while shuffling data to simulate null effect and createa sampling distribution.</strong></li>
<li>Calculate p-value.</li>
<li>Draw conclusion.</li>
</ul>
</li>
</ul>
<p>To conclude, analytical method often dictates the test statistic, and tests have assumptions we must meet. This is not the case while simulating. In addition, if we have issues like censored data, non-independence, or long-tailed distributions, we may not find an off-the-shelf test. In such a case, unless we are mathematical statisticians, we may not be able to make a suitable statistical test by ourselves. Computational approach seems to be more easy to comprehend and to execute.</p>
